# ローカルサーバー接続問題 - トラブルシューティングガイド

## 発生した問題

### 症状
- ブラウザで `localhost:8080` にアクセス時に「このサイトにアクセスできません」エラー
- `ERR_CONNECTION_REFUSED` エラーコードが表示
- 「localhost で接続が拒否されました。」メッセージ

### 発生日時
2025年7月23日 6:25頃

### スクリーンショット
`/Users/apple/Desktop/スクリーンショット 2025-07-23 6.25.10.png` に記録済み

## 原因分析

### 根本原因
Python HTTPサーバーが起動していない状態だった

### 調査手順
1. `ps aux | grep python` でPythonプロセス確認 → サーバープロセス無し
2. `lsof -i :8080` でポート使用状況確認 → ポート8080未使用
3. プロジェクトファイル確認 → `index.html`, `css/`, `js/` ファイル正常存在

### 結論
単純にローカルサーバーが未起動の状態でアクセスを試みたことが原因

## 解決方法

### 即座の対処
```bash
# プロジェクトディレクトリに移動
cd /Users/apple/Desktop/開発/HomePage2

# Python HTTPサーバーをバックグラウンドで起動
python3 -m http.server 8080 &
```

### 確認コマンド
```bash
# サーバー起動確認
ps aux | grep 8080

# HTTPレスポンス確認
curl -I http://localhost:8080/
```

### 期待する結果
```
HTTP/1.0 200 OK
Server: SimpleHTTP/0.6 Python/3.9.6
Content-type: text/html
Content-Length: 4523
```

## 予防策

### 開発時の推奨手順
1. プロジェクト開始時に必ずローカルサーバーを起動
2. サーバー状況を定期的に確認
3. 開発終了時はサーバーを適切に停止

### サーバー管理コマンド
```bash
# サーバー起動
python3 -m http.server 8080

# バックグラウンド起動
python3 -m http.server 8080 &

# サーバー停止（プロセスID確認後）
kill [PID]
```
